臺股開盤前風向提醒機器人 1.0版本創設日誌
一、目標簡述
功能：每日彙整臺股大盤重要技術指標並一次推播，降低監控負擔並協助交易決策。
願景：透過自動化與指標量化，協助邁向長期財務目標。
二 指標說明與解讀
    1.券資比 (TWSE)
        說明：衡量當日融券（做空）與融資（槓桿買進）行為相對強弱的指標。
        為何可觀察大盤：券資比變動反映市場槓桿與多空偏好；短期內大幅上升可能代表做空力量累積或融資縮手（偏空訊號），但在過度做空情況下則可能觸發回補，成為反向警訊。

    2.臺指選擇權波動率 (VIX)
        說明：俗稱臺股的'恐懼指標'，由選擇權隱含波動率計算的指標，數值越高表示市場預期未來波動越大，也代表情緒緊張。
        為何可觀察大盤：VIX 上升通常伴隨避險需求增加，抑制追價並可能導致短期回檔；VIX 下降則代表風險偏好回升，有利多方延續或反彈。

    3.臺股期貨未平倉口數（外資 / 自營商 / 投信）
        說明：顯示各類參與者持有的期貨淨部位量，正負或增減代表押注方向與規模變化。
        為何可觀察大盤：外資、大型交易方部位變化往往與資金流向相關：外資做空可能對指數造成壓力；國內投信、自營若擴大多單，則提供短中期支撐。綜合三方動向可看出趨勢延續或轉折可能。

    4.所有融資上市股票市值
        說明：市場中以融資方式持有之股票總市值或融資餘額總和，代表槓桿資金的參與規模。
        為何可觀察大盤：高槓桿參與可放大行情：上漲時擴大漲幅，但下跌時會因強制平倉加速回檔；若融資市值快速萎縮，可能提示市場信心流失或已出現強制減碼風險。

    5.大盤融資維持率
        說明：衡量整體融資戶維持保證金比例的指標，數值越高表示抗跌能力越強。
        為何可觀察大盤：維持率下降至低檔易引發保證金追繳與強制平倉(俗稱斷頭)，造成價格加速下跌；維持率穩健則有助於市場穩定。

三、資料取得與設計

    1.券資比:使用臺灣證券交易所API回傳資料後計算得出結果。
    2.臺指選擇權波動率 (VIX):採用網路爬蟲取得回傳內容。
    3.臺股期貨未平倉口數:採用網路爬蟲取得回傳內容。
    4.所有融資上市股票市值:使用臺灣證券交易所API回傳全部上市股票收盤資料、融資、融券張數，將個股收盤價、融資張數相乘得出。
    5.大盤融資維持率:大盤融資維持率的公式為(所有融資股票市值／大盤融資餘額)，大盤融資餘額採網路爬蟲取得資料，後與前述得到的所有融資股票市值相除而得。

四、運行方式與進行時間

使用google cloud雲端運行，時間為每日00:02，如當天非開盤日，就不會運行。

五、相關疑問

    1.為什麼選擇這五個指標:這五項指標是近幾年操作臺股開盤前，都會參考的指標，主要是透過這些指標能理解今日臺股的大致走勢。避免追高殺低，但這些指標分散在不同網站，故透過程式碼將其蒐集後一次推播。
    2.運行時間的選擇:相關資料皆為當日收盤後才會更新的資料，因此設每日更新，因更新仍須時間，且每次資料回傳時間不一，故設於00:02運行程式。
    3.大盤融資維持率數值怎麼與券商、民間網站有些不同:大盤融資維持率並沒有在證交所等官方網站上更新，需要自行計算，先前已比對多家券商，參考不同網站的計算方式，已排除暫停融資、融券、停止交易的股票，大致跟券商會有1%-2%的誤差，但數值經過多次驗算,已為正確數值；期間發現某些民間網站的計算方式有些疑慮，仍須小心。

臺股開盤前風向提醒機器人 2.0版本更新日誌
一、本次更新目標

改善大盤融資維持率的計算誤差，並讓推播內容更易讀、資訊更完整。

二、指標來源變更

    1.大盤融資維持率/融資融券:不再自行用個股融資張數×收盤價計算，改為直接抓取 CMoney（融資融券頁面）已計算好的數值，包含融資餘額、增減、使用率、維持率，以及融券餘額、增減、使用率。融資（金額）與融券（張數）單位不同，呈現時不混用、不相除。
    2.新增台指期夜盤指數:抓取 CMoney 台指期夜盤頁面，顯示夜盤指數、漲跌、漲跌幅，作為早盤前對隔夜風險的參考。
    3.原本的個股加總計算邏輯（maintenance_calc）保留在程式碼中但不再預設執行，可用 ORDERED_SCRAPERS 環境變數手動啟用。

三、推播格式改版

    1.訊息改為分區呈現（大盤指數／市場情緒／券資比／期貨未平倉口數／融資融券），並加入 emoji 與燈號標示，手機閱讀更直覺。
    2.VIX 解讀方式改為依「與前一交易日的變化幅度」判斷市場情緒（急升/升溫/平穩/回落/明顯消退），而非單看絕對值，因近期大盤位於高點，VIX absolute 水位已不適合作為唯一判斷依據。
    3.融資維持率燈號分為四段：≥180% 過高、150-180% 中等、130-150% 偏低、<130% 危險（130%為一般追繳水位參考）。
    4.期貨未平倉口數直接顯示外資/自營商/投信的口數與較前一交易日的變化量。

四、運行方式變更

    1.推播時間由每日00:02改為每日08:00（開盤前），資料仍為前一交易日收盤後數據，定位為開盤前提醒。
    2.新增 GCS（Google Cloud Storage）儲存歷史資料：Cloud Run Job 每次執行皆為全新容器、本機磁碟不會保留，故新增雲端儲存桶同步每日存檔，讓"與前一交易日比較"的邏輯在雲端也能正確運作。
    3.Docker image 預設安裝 Playwright 瀏覽器（支援新增的 CMoney 爬蟲）。

臺股開盤前風向提醒機器人 2.1版本更新日誌
一、本次更新目標

修復 VIX 顯示異常值的問題，並新增美股恐懼貪婪指數、台積電 ADR 比較、AI 簡評三項指標，讓推播更完整、更易判讀。

二、Bug 修復

    1.修復 VIX 顯示 0 的問題:VIXTWN 來源 API 偶爾會回傳無效的「0」佔位值，先前程式碼會將其誤判為合法收盤值並覆蓋快取。現已改為將數值 <=0 視為錯誤、不再覆蓋快取；compose_notification 在當次抓取失敗或缺席時，會自動 fallback 讀取前次有效的快取值，並標註「（快取）」。

三、新增指標

    1.美股恐懼貪婪指數（CNN Fear & Greed Index）:呼叫 CNN 背後的資料 API（production.dataviz.cnn.io），依分數分為極度恐懼、恐懼、中性、貪婪、極度貪婪五級。
    2.台積電 ADR 與台股 2330 隱含溢價/折價:以 yfinance 取得台積電紐約 ADR（TSM，1 ADR：5 股）與台股 2330 前一交易日收盤價，換算美元兌台幣匯率後比較，得出隱含溢價或折價百分比。
    3.AI 簡評:彙整當日夜盤、VIX、美股恐懼貪婪指數、券資比、期貨未平倉、融資融券等關鍵數據，呼叫 Claude API（claude-haiku-4-5）生成一段約 50 字的三句式中文簡評，放在推播訊息最上方。未設定 ANTHROPIC_API_KEY 或 API 呼叫失敗時自動跳過，不影響其他指標正常推播。

四、推播格式調整

    1.訊息區塊順序調整為：(AI 簡評) → 大盤指數 → 市場情緒(VIX) → 美股恐懼貪婪指數 → 台積電 ADR → 券資比 → 期貨未平倉口數 → 融資融券。
    2.台指期夜盤的漲跌幅文字說明調整為「較前一日收盤 +0.36%」，避免誤會百分比基準。

五、雲端部署變更

    1.新增 ANTHROPIC_API_KEY 至 GCP Secret Manager，並授權既有的 tw-stock-runner 服務帳戶讀取，沿用與 TG_BOT_TOKEN/TG_CHAT_ID 相同的金鑰管理方式，金鑰不會出現在程式碼或映像檔中。
    2.重新建置並推送 Docker image 至 gcr.io/twseaftertradingdetect/tw_stock:latest，並更新既有 Cloud Run Job 設定加入新的 secret 環境變數。
